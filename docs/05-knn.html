5.2 Introducción

En el capítulo anterior estudiamos la regresión logística, un modelo supervisado que estima probabilidades para resolver problemas de clasificación binaria.

En este capítulo estudiaremos otro método de clasificación muy intuitivo: k vecinos más cercanos, conocido como k-NN por sus siglas en inglés: k-Nearest Neighbors.

La idea básica de k-NN es sencilla:

Para clasificar una nueva observación, buscamos las observaciones más parecidas a ella y le asignamos la clase más frecuente entre sus vecinos.

Por ejemplo, si queremos clasificar un nuevo accidente de tránsito, el algoritmo busca accidentes anteriores con características similares. Si la mayoría de esos accidentes similares fueron de “Solo daños”, entonces clasificará el nuevo accidente como “Solo daños”. Si la mayoría fueron “Con víctimas”, entonces lo clasificará como “Con víctimas”.

5.3 Idea intuitiva del método

Supongamos que tenemos un nuevo accidente que queremos clasificar.

El algoritmo k-NN hace lo siguiente:

  1. Calcula la distancia entre el nuevo accidente y los accidentes históricos.
  2. Encuentra los (k) accidentes más cercanos.
  3. Revisa la clase de esos vecinos.
  4. Asigna la clase más frecuente.

Por ejemplo, si usamos:

k = 5

y los cinco vecinos más cercanos tienen estas clases:

Solo daños
Solo daños
Con víctimas
Solo daños
Solo daños

entonces la clase más frecuente es:

Solo daños

Por lo tanto, el nuevo accidente se clasifica como:

Solo daños

5.4 ¿Qué significa k?

La letra (k) indica cuántos vecinos se usarán para clasificar una observación.

Algunos ejemplos:

k = 1   → se usa el vecino más cercano
k = 3   → se usan los 3 vecinos más cercanos
k = 5   → se usan los 5 vecinos más cercanos
k = 11  → se usan los 11 vecinos más cercanos

El valor de (k) es importante.

Si (k) es muy pequeño, el modelo puede ser muy sensible al ruido.

Si (k) es muy grande, el modelo puede volverse demasiado general y perder detalles locales.

5.6 Distancia euclidiana

k-NN necesita medir qué tan parecidas son dos observaciones.

Una forma común de hacerlo es mediante la distancia euclidiana.

Para dos puntos:

[ A = (x_1, y_1) ]

y

[ B = (x_2, y_2) ]

la distancia euclidiana es:

[ d(A,B) = ]

Esta fórmula mide la distancia en línea recta entre dos puntos.

5.7 Ejemplo de cálculo de distancia

Calculemos la distancia entre dos accidentes.

El primer accidente ocurrió a las 8 horas en el mes 1.

El segundo ocurrió a las 10 horas en el mes 2.

Code 
x1 <- 8
y1 <- 1

x2 <- 10
y2 <- 2

distancia <- sqrt((x1 - x2)^2 + (y1 - y2)^2)

distancia
[1] 2.236068

La distancia indica qué tan separados están los dos accidentes en el espacio definido por las variables hora y mes.

5.8 ¿Por qué hay que escalar las variables?

k-NN depende de distancias. Por eso, la escala de las variables importa mucho.

Supongamos que usamos dos variables:

edad
ingreso mensual

La edad puede variar entre 18 y 90, mientras que el ingreso puede variar entre 5,000 y 100,000.

Si usamos distancias directamente, la variable ingreso dominará el cálculo, simplemente porque tiene números más grandes.

Para evitar este problema, normalmente estandarizamos las variables.

Una forma común de estandarizar es transformar cada variable para que tenga:

media 0
desviación estándar 1

En R podemos hacerlo con la función:

scale()

5.9 k-NN con variables categóricas

El método k-NN funciona naturalmente con variables numéricas, porque se basa en distancias.

Pero muchas bases reales, como ATUS, tienen variables categóricas:

DIASEMANA
TIPACCID
CAUSAACCI

Para usar k-NN con estas variables, necesitamos transformarlas en variables numéricas.

Una estrategia común es crear variables indicadoras, también llamadas variables dummy.

Por ejemplo, si tenemos la variable:

DIASEMANA

con valores:

lunes
Martes
Miercoles

podemos crear columnas como:

DIASEMANA_lunes
DIASEMANA_Martes
DIASEMANA_Miercoles

Cada columna toma el valor 1 si la observación pertenece a esa categoría y 0 en caso contrario.

5.17 Estandarización de variables

Como k-NN depende de distancias, debemos estandarizar las variables.

Es importante calcular la media y desviación estándar usando solo el conjunto de entrenamiento.

Después aplicamos esa misma transformación al conjunto de prueba.

Code 
if (exists("x_entrenamiento") && exists("x_prueba")) {
  medias <- apply(x_entrenamiento, 2, mean)
  desviaciones <- apply(x_entrenamiento, 2, sd)
  
  desviaciones[desviaciones == 0] <- 1
  
  x_entrenamiento_esc <- scale(
    x_entrenamiento,
    center = medias,
    scale = desviaciones
  )
  
  x_prueba_esc <- scale(
    x_prueba,
    center = medias,
    scale = desviaciones
  )
}

Esta forma de escalar evita que usemos información del conjunto de prueba durante el entrenamiento.

Eso es importante porque el conjunto de prueba debe simular datos nuevos.

5.21 Interpretación de las métricas

La exactitud indica la proporción total de clasificaciones correctas.

La sensibilidad indica qué proporción de accidentes con víctimas fueron detectados correctamente.

La especificidad indica qué proporción de accidentes de solo daños fueron clasificados correctamente.

Como ya vimos en el capítulo anterior, esta base está desbalanceada. Por eso no debemos evaluar el modelo solo con exactitud.

En seguridad vial, la sensibilidad puede ser especialmente importante, porque nos interesa detectar accidentes con víctimas.

5.22 Comparación con distintos valores de k

El valor de (k) puede afectar los resultados.

Probaremos tres valores:

k = 3
k = 5
k = 11

Primero creamos una función para calcular métricas.

Code 
calcular_metricas <- function(real, predicho) {
  
  real <- factor(real, levels = c("Con víctimas", "Solo daños"))
  predicho <- factor(predicho, levels = c("Con víctimas", "Solo daños"))
  
  matriz <- table(
    Real = real,
    Predicho = predicho
  )
  
  VP <- matriz["Con víctimas", "Con víctimas"]
  FN <- matriz["Con víctimas", "Solo daños"]
  FP <- matriz["Solo daños", "Con víctimas"]
  VN <- matriz["Solo daños", "Solo daños"]
  
  exactitud <- (VP + VN) / (VP + FN + FP + VN)
  sensibilidad <- VP / (VP + FN)
  especificidad <- VN / (VN + FP)
  
  data.frame(
    exactitud = exactitud,
    sensibilidad = sensibilidad,
    especificidad = especificidad
  )
}

Ahora probamos varios valores de (k).

Code 
if (exists("x_entrenamiento_esc") && exists("x_prueba_esc")) {
  
  valores_k <- c(3, 5, 11)
  
  resultados_knn <- data.frame()
  
  for (k_actual in valores_k) {

      train = x_entrenamiento_esc,
      test = x_prueba_esc,

5.24 Interpretación de la comparación

El valor de (k) influye en el comportamiento del modelo.

Con valores pequeños de (k), el modelo puede reaccionar mucho a observaciones individuales. Esto puede hacerlo sensible al ruido.

Con valores grandes de (k), el modelo toma decisiones más suaves, pero puede perder capacidad para detectar patrones locales.

No existe un valor de (k) universalmente mejor. Debe elegirse evaluando el desempeño del modelo y considerando el objetivo del problema.

En este caso, como el problema está desbalanceado, no basta con elegir el valor de (k) que tenga mayor exactitud. También debemos revisar sensibilidad y especificidad.

5.25 Comparación conceptual con regresión logística

La regresión logística y k-NN son modelos de clasificación, pero funcionan de manera muy diferente.

Aspecto Regresión logística k-NN
Tipo de modelo Paramétrico No paramétrico
Produce modelo explícito Sí No en el mismo sentido
Usa distancias No Sí
Requiere escalamiento No siempre Sí
Interpretable Alta Media o baja
Sensible a muchas variables Moderadamente Sí
Costo de predicción Bajo Puede ser alto
Manejo natural de categóricas Sí, con factores Requiere transformación

La regresión logística estima una relación matemática entre variables y probabilidad.

k-NN clasifica comparando cada nueva observación contra observaciones existentes.

5.26 Ventajas de k-NN

Algunas ventajas de k-NN son:

  • es fácil de entender;
  • no requiere asumir una forma funcional específica;
  • puede capturar fronteras de decisión no lineales;
  • es útil como método introductorio de clasificación;
  • permite explicar visualmente la idea de cercanía y similitud.

5.27 Limitaciones de k-NN

k-NN también tiene limitaciones importantes:

  • puede ser lento con bases grandes;
  • depende mucho de la escala de las variables;
  • puede verse afectado por variables irrelevantes;
  • no maneja variables categóricas directamente;
  • requiere elegir el valor de (k);
  • puede tener dificultades en espacios de muchas dimensiones;
  • no produce una ecuación interpretable como la regresión logística.

En el caso de ATUS, usamos variables categóricas transformadas en variables dummy. Esto permite aplicar k-NN, pero también aumenta el número de columnas.

5.28 ¿Qué aprendimos?

En este capítulo aprendimos que:

  • k-NN clasifica una observación según sus vecinos más cercanos;
  • el parámetro (k) indica cuántos vecinos se consideran;
  • el método depende de una medida de distancia;
  • las variables deben estar en una escala comparable;
  • las variables categóricas deben transformarse antes de usar k-NN;
  • el desempeño cambia según el valor de (k);
  • en bases desbalanceadas debemos revisar más que la exactitud.

5.29 Conclusión

k-NN es uno de los métodos más intuitivos del aprendizaje automático.

Aunque es sencillo de explicar, requiere cuidado en la preparación de los datos, especialmente en el escalamiento y en el tratamiento de variables categóricas.

En este capítulo usamos k-NN para clasificar accidentes de tránsito según la presencia de víctimas.

## Introducción

En el capítulo anterior estudiamos la regresión logística, un modelo supervisado que estima probabilidades para resolver problemas de clasificación binaria.

En este capítulo estudiaremos otro método de clasificación muy intuitivo: **k vecinos más cercanos**, conocido como **k-NN** por sus siglas en inglés: *k-Nearest Neighbors*.

La idea básica de k-NN es sencilla:

> Para clasificar una nueva observación, buscamos las observaciones más parecidas a ella y le asignamos la clase más frecuente entre sus vecinos.

Por ejemplo, si queremos clasificar un nuevo accidente de tránsito, el algoritmo busca accidentes anteriores con características similares. Si la mayoría de esos accidentes similares fueron de “Solo daños”, entonces clasificará el nuevo accidente como “Solo daños”. Si la mayoría fueron “Con víctimas”, entonces lo clasificará como “Con víctimas”.

## Idea intuitiva del método

Supongamos que tenemos un nuevo accidente que queremos clasificar.

El algoritmo k-NN hace lo siguiente:

1. Calcula la distancia entre el nuevo accidente y los accidentes históricos.
2. Encuentra los \(k\) accidentes más cercanos.
3. Revisa la clase de esos vecinos.
4. Asigna la clase más frecuente.

Por ejemplo, si usamos:

```text
k = 5
```

y los cinco vecinos más cercanos tienen estas clases:

```text
Solo daños
Solo daños
Con víctimas
Solo daños
Solo daños
```

entonces la clase más frecuente es:

```text
Solo daños
```

Por lo tanto, el nuevo accidente se clasifica como:

```text
Solo daños
```

## ¿Qué significa k?

La letra \(k\) indica cuántos vecinos se usarán para clasificar una observación.

Algunos ejemplos:

```text
k = 1   → se usa el vecino más cercano
k = 3   → se usan los 3 vecinos más cercanos
k = 5   → se usan los 5 vecinos más cercanos
k = 11  → se usan los 11 vecinos más cercanos
```

El valor de \(k\) es importante.

Si \(k\) es muy pequeño, el modelo puede ser muy sensible al ruido.

Si \(k\) es muy grande, el modelo puede volverse demasiado general y perder detalles locales.

## Distancia euclidiana

k-NN necesita medir qué tan parecidas son dos observaciones.

Una forma común de hacerlo es mediante la **distancia euclidiana**.

Para dos puntos:

\[
A = (x_1, y_1)
\]

y

\[
B = (x_2, y_2)
\]

la distancia euclidiana es:

\[
d(A,B) = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2}
\]

Esta fórmula mide la distancia en línea recta entre dos puntos.

## Ejemplo de cálculo de distancia

Calculemos la distancia entre dos accidentes.

El primer accidente ocurrió a las 8 horas en el mes 1.

El segundo ocurrió a las 10 horas en el mes 2.

```{r}
x1 <- 8
y1 <- 1

x2 <- 10
y2 <- 2

distancia <- sqrt((x1 - x2)^2 + (y1 - y2)^2)

distancia
```

La distancia indica qué tan separados están los dos accidentes en el espacio definido por las variables hora y mes.

## ¿Por qué hay que escalar las variables?

k-NN depende de distancias. Por eso, la escala de las variables importa mucho.

Supongamos que usamos dos variables:

```text
edad
ingreso mensual
```

La edad puede variar entre 18 y 90, mientras que el ingreso puede variar entre 5,000 y 100,000.

Si usamos distancias directamente, la variable ingreso dominará el cálculo, simplemente porque tiene números más grandes.

Para evitar este problema, normalmente estandarizamos las variables.

Una forma común de estandarizar es transformar cada variable para que tenga:

```text
media 0
desviación estándar 1
```

En R podemos hacerlo con la función:

```r
scale()
```

## k-NN con variables categóricas

El método k-NN funciona naturalmente con variables numéricas, porque se basa en distancias.

Pero muchas bases reales, como ATUS, tienen variables categóricas:

```text
DIASEMANA
TIPACCID
CAUSAACCI
```

Para usar k-NN con estas variables, necesitamos transformarlas en variables numéricas.

Una estrategia común es crear variables indicadoras, también llamadas variables *dummy*.

Por ejemplo, si tenemos la variable:

```text
DIASEMANA
```

con valores:

```text
lunes
Martes
Miercoles
```

podemos crear columnas como:

```text
DIASEMANA_lunes
DIASEMANA_Martes
DIASEMANA_Miercoles
```

Cada columna toma el valor 1 si la observación pertenece a esa categoría y 0 en caso contrario.

## Comparación con distintos valores de k

El valor de \(k\) puede afectar los resultados.

Probaremos tres valores:

```text
k = 3
k = 5
k = 11
```

Primero creamos una función para calcular métricas.

```{r}
calcular_metricas <- function(real, predicho) {
  
  real <- factor(real, levels = c("Con víctimas", "Solo daños"))
  predicho <- factor(predicho, levels = c("Con víctimas", "Solo daños"))
  
  matriz <- table(
    Real = real,
    Predicho = predicho
  )
  
  VP <- matriz["Con víctimas", "Con víctimas"]
  FN <- matriz["Con víctimas", "Solo daños"]
  FP <- matriz["Solo daños", "Con víctimas"]
  VN <- matriz["Solo daños", "Solo daños"]
  
  exactitud <- (VP + VN) / (VP + FN + FP + VN)
  sensibilidad <- VP / (VP + FN)
  especificidad <- VN / (VN + FP)
  
  data.frame(
    exactitud = exactitud,
    sensibilidad = sensibilidad,
    especificidad = especificidad
  )
}
```

Ahora probamos varios valores de \(k\).

```{r}
if (exists("x_entrenamiento_esc") && exists("x_prueba_esc")) {
  
  valores_k <- c(3, 5, 11)
  
  resultados_knn <- data.frame()
  
  for (k_actual in valores_k) {
    
    pred_actual <- knn(
      train = x_entrenamiento_esc,
      test = x_prueba_esc,
      cl = y_entrenamiento,
      k = k_actual
    )
    
    metricas_actuales <- calcular_metricas(
      real = y_prueba,
      predicho = pred_actual
    )
    
    metricas_actuales$k <- k_actual
    
    resultados_knn <- rbind(
      resultados_knn,
      metricas_actuales
    )
  }
  
  resultados_knn <- resultados_knn |>
    select(k, exactitud, sensibilidad, especificidad)
  
  resultados_knn
}
```

## Interpretación de la comparación

El valor de \(k\) influye en el comportamiento del modelo.

Con valores pequeños de \(k\), el modelo puede reaccionar mucho a observaciones individuales. Esto puede hacerlo sensible al ruido.

Con valores grandes de \(k\), el modelo toma decisiones más suaves, pero puede perder capacidad para detectar patrones locales.

No existe un valor de \(k\) universalmente mejor. Debe elegirse evaluando el desempeño del modelo y considerando el objetivo del problema.

En este caso, como el problema está desbalanceado, no basta con elegir el valor de \(k\) que tenga mayor exactitud. También debemos revisar sensibilidad y especificidad.

## Comparación conceptual con regresión logística

La regresión logística y k-NN son modelos de clasificación, pero funcionan de manera muy diferente.

| Aspecto | Regresión logística | k-NN |
|---|---|---|
| Tipo de modelo | Paramétrico | No paramétrico |
| Produce modelo explícito | Sí | No en el mismo sentido |
| Usa distancias | No | Sí |
| Requiere escalamiento | No siempre | Sí |
| Interpretable | Alta | Media o baja |
| Sensible a muchas variables | Moderadamente | Sí |
| Costo de predicción | Bajo | Puede ser alto |
| Manejo natural de categóricas | Sí, con factores | Requiere transformación |

La regresión logística estima una relación matemática entre variables y probabilidad.

k-NN clasifica comparando cada nueva observación contra observaciones existentes.

## Ventajas de k-NN

Algunas ventajas de k-NN son:

- es fácil de entender;
- no requiere asumir una forma funcional específica;
- puede capturar fronteras de decisión no lineales;
- es útil como método introductorio de clasificación;
- permite explicar visualmente la idea de cercanía y similitud.

## Limitaciones de k-NN

k-NN también tiene limitaciones importantes:

- puede ser lento con bases grandes;
- depende mucho de la escala de las variables;
- puede verse afectado por variables irrelevantes;
- no maneja variables categóricas directamente;
- requiere elegir el valor de \(k\);
- puede tener dificultades en espacios de muchas dimensiones;
- no produce una ecuación interpretable como la regresión logística.

En el caso de ATUS, usamos variables categóricas transformadas en variables dummy. Esto permite aplicar k-NN, pero también aumenta el número de columnas.

## ¿Qué aprendimos?

En este capítulo aprendimos que:

- k-NN clasifica una observación según sus vecinos más cercanos;
- el parámetro \(k\) indica cuántos vecinos se consideran;
- el método depende de una medida de distancia;
- las variables deben estar en una escala comparable;
- las variables categóricas deben transformarse antes de usar k-NN;
- el desempeño cambia según el valor de \(k\);
- en bases desbalanceadas debemos revisar más que la exactitud.

## Conclusión

k-NN es uno de los métodos más intuitivos del aprendizaje automático.

Aunque es sencillo de explicar, requiere cuidado en la preparación de los datos, especialmente en el escalamiento y en el tratamiento de variables categóricas.

En este capítulo usamos k-NN para clasificar accidentes de tránsito según la presencia de víctimas.
