5.2 Introducción

k-NN significa k-Nearest Neighbors o k vecinos más cercanos.

La idea básica es:

Para clasificar una nueva observación, buscamos las observaciones más parecidas a ella y le asignamos la clase más frecuente entre sus vecinos.

5.3 Idea intuitiva del método

El algoritmo k-NN hace lo siguiente:

  1. Calcula la distancia entre la nueva observación y las observaciones históricas.
  2. Encuentra los \(k\) vecinos más cercanos.
  3. Revisa la clase de esos vecinos.
  4. Asigna la clase más frecuente.

5.4 ¿Qué significa k?

La letra \(k\) indica cuántos vecinos se usarán para clasificar una observación.

Si \(k\) es muy pequeño, el modelo puede ser muy sensible al ruido. Si \(k\) es muy grande, el modelo puede volverse demasiado general.

5.6 Distancia euclidiana

Para dos puntos:

\[ A = (x_1, y_1) \]

y

\[ B = (x_2, y_2) \]

la distancia euclidiana es:

\[ d(A,B) = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2} \]

5.7 Fundamento matemático de k-NN

Sea \(x_0\) una nueva observación que queremos clasificar.

Sea \(x_i\) una observación del conjunto de entrenamiento.

Si tenemos \(p\) variables predictoras, podemos escribir:

\[ x_0 = (x_{01}, x_{02}, \ldots, x_{0p}) \]

y

\[ x_i = (x_{i1}, x_{i2}, \ldots, x_{ip}) \]

La distancia euclidiana entre \(x_0\) y \(x_i\) se define como:

\[ d(x_0, x_i) = \sqrt{ \sum_{j=1}^{p} (x_{0j} - x_{ij})^2 } \]

Después se seleccionan las \(k\) observaciones con menor distancia.

Si las clases de los \(k\) vecinos más cercanos son:

\[ y_{(1)}, y_{(2)}, \ldots, y_{(k)} \]

entonces la clase predicha se obtiene por mayoría de votos:

\[ \hat{y}_0 = \operatorname{moda} \left( y_{(1)}, y_{(2)}, \ldots, y_{(k)} \right) \]

5.9 Estandarización de variables

Como k-NN depende de distancias, es importante que las variables estén en escalas comparables.

Una forma común de estandarizar una variable \(X_j\) es:

\[ z_{ij} = \frac{x_{ij} - \bar{x}_j}{s_j} \]

donde \(\bar{x}_j\) es la media de la variable \(j\) y \(s_j\) es su desviación estándar.

5.10 Algoritmo k-NN

Algoritmo: k vecinos más cercanos

Entrada:
  - Matriz de entrenamiento X
  - Clases de entrenamiento Y
  - Nueva observación x_0
  - Número de vecinos k

Proceso:
  1. Estandarizar las variables predictoras.
  2. Calcular la distancia entre x_0 y cada observación de entrenamiento.
  3. Ordenar las distancias de menor a mayor.
  4. Seleccionar los k vecinos más cercanos.
  5. Revisar las clases de esos k vecinos.
  6. Contar cuántas veces aparece cada clase.
  7. Asignar a x_0 la clase más frecuente.

Salida:
  - Clase predicha para x_0

calcular_metricas <- function(real, predicho) {
  real <- factor(real, levels = c("Con víctimas", "Solo daños"))
  predicho <- factor(predicho, levels = c("Con víctimas", "Solo daños"))

  matriz <- table(Real = real, Predicho = predicho)

  VP <- matriz["Con víctimas", "Con víctimas"]
  FN <- matriz["Con víctimas", "Solo daños"]
  FP <- matriz["Solo daños", "Con víctimas"]
  VN <- matriz["Solo daños", "Solo daños"]

  exactitud <- (VP + VN) / (VP + FN + FP + VN)
  sensibilidad <- VP / (VP + FN)
  especificidad <- VN / (VN + FP)

  data.frame(exactitud = exactitud, sensibilidad = sensibilidad, especificidad = especificidad)
}

## Introducción

k-NN significa *k-Nearest Neighbors* o **k vecinos más cercanos**.

La idea básica es:

> Para clasificar una nueva observación, buscamos las observaciones más parecidas a ella y le asignamos la clase más frecuente entre sus vecinos.

## Idea intuitiva del método

El algoritmo k-NN hace lo siguiente:

1. Calcula la distancia entre la nueva observación y las observaciones históricas.
2. Encuentra los $k$ vecinos más cercanos.
3. Revisa la clase de esos vecinos.
4. Asigna la clase más frecuente.

## ¿Qué significa k?

La letra $k$ indica cuántos vecinos se usarán para clasificar una observación.

Si $k$ es muy pequeño, el modelo puede ser muy sensible al ruido. Si $k$ es muy grande, el modelo puede volverse demasiado general.

## Distancia euclidiana

Para dos puntos:

$$
A = (x_1, y_1)
$$

y

$$
B = (x_2, y_2)
$$

la distancia euclidiana es:

$$
d(A,B) = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2}
$$

## Fundamento matemático de k-NN

Sea $x_0$ una nueva observación que queremos clasificar.

Sea $x_i$ una observación del conjunto de entrenamiento.

Si tenemos $p$ variables predictoras, podemos escribir:

$$
x_0 = (x_{01}, x_{02}, \ldots, x_{0p})
$$

y

$$
x_i = (x_{i1}, x_{i2}, \ldots, x_{ip})
$$

La distancia euclidiana entre $x_0$ y $x_i$ se define como:

$$
d(x_0, x_i)
=
\sqrt{
\sum_{j=1}^{p}
(x_{0j} - x_{ij})^2
}
$$

Después se seleccionan las $k$ observaciones con menor distancia.

Si las clases de los $k$ vecinos más cercanos son:

$$
y_{(1)}, y_{(2)}, \ldots, y_{(k)}
$$

entonces la clase predicha se obtiene por mayoría de votos:

$$
\hat{y}_0 =
\operatorname{moda}
\left(
y_{(1)}, y_{(2)}, \ldots, y_{(k)}
\right)
$$

## Estandarización de variables

Como k-NN depende de distancias, es importante que las variables estén en escalas comparables.

Una forma común de estandarizar una variable $X_j$ es:

$$
z_{ij}
=
\frac{x_{ij} - \bar{x}_j}{s_j}
$$

donde $\bar{x}_j$ es la media de la variable $j$ y $s_j$ es su desviación estándar.

## Algoritmo k-NN

```text
Algoritmo: k vecinos más cercanos

Entrada:
  - Matriz de entrenamiento X
  - Clases de entrenamiento Y
  - Nueva observación x_0
  - Número de vecinos k

Proceso:
  1. Estandarizar las variables predictoras.
  2. Calcular la distancia entre x_0 y cada observación de entrenamiento.
  3. Ordenar las distancias de menor a mayor.
  4. Seleccionar los k vecinos más cercanos.
  5. Revisar las clases de esos k vecinos.
  6. Contar cuántas veces aparece cada clase.
  7. Asignar a x_0 la clase más frecuente.

Salida:
  - Clase predicha para x_0
```

calcular_metricas <- function(real, predicho) {
  real <- factor(real, levels = c("Con víctimas", "Solo daños"))
  predicho <- factor(predicho, levels = c("Con víctimas", "Solo daños"))

  matriz <- table(Real = real, Predicho = predicho)

  VP <- matriz["Con víctimas", "Con víctimas"]
  FN <- matriz["Con víctimas", "Solo daños"]
  FP <- matriz["Solo daños", "Con víctimas"]
  VN <- matriz["Solo daños", "Solo daños"]

  exactitud <- (VP + VN) / (VP + FN + FP + VN)
  sensibilidad <- VP / (VP + FN)
  especificidad <- VN / (VN + FP)

  data.frame(exactitud = exactitud, sensibilidad = sensibilidad, especificidad = especificidad)
