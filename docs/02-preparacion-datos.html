2.1 Objetivos del capítulo

Al finalizar este capítulo, el lector será capaz de:

  • Comprender por qué la preparación de datos es una etapa fundamental en aprendizaje automático.
  • Leer un conjunto de datos real en R.
  • Revisar nombres de variables, dimensiones y estructura de una base.
  • Seleccionar variables de interés.
  • Crear una variable respuesta para un problema de clasificación.
  • Preparar una primera base de trabajo lista para análisis exploratorio y modelado.
  • Representar matemáticamente una base preparada para aprendizaje automático.

2.2 Introducción

En los libros y cursos introductorios es común usar bases de datos pequeñas, limpias y listas para modelar. Sin embargo, en la práctica profesional los datos reales casi nunca vienen preparados.

Antes de aplicar cualquier algoritmo de aprendizaje automático, normalmente tenemos que realizar varias tareas:

  • descargar o importar datos,
  • revisar nombres de variables,
  • corregir tipos de datos,
  • detectar valores faltantes,
  • seleccionar variables útiles,
  • transformar variables,
  • crear una variable respuesta,
  • eliminar registros incompletos,
  • documentar cada paso.

Esta etapa se conoce como preparación de datos.

En muchos proyectos reales, preparar los datos toma más tiempo que entrenar el modelo.

2.3 ¿Por qué es tan importante preparar los datos?

Un modelo de aprendizaje automático no puede corregir por sí solo todos los problemas de una base de datos.

Si los datos están mal preparados, el modelo puede producir resultados equivocados, aunque el algoritmo sea sofisticado.

Una frase común en ciencia de datos es:

Basura entra, basura sale.

Esto significa que si usamos datos mal organizados, incompletos o incorrectos, el resultado del modelo también será poco confiable.

2.4 Datos crudos y datos preparados

Llamaremos datos crudos a la base tal como se obtiene de la fuente original.

Los datos crudos son valiosos porque contienen la información original, pero no siempre están listos para aplicar aprendizaje automático.

Llamaremos datos preparados a la base que ya fue revisada, transformada y organizada para un propósito específico.

La diferencia puede verse así:

Datos crudos
    ↓
Revisión
    ↓
Limpieza
    ↓
Selección de variables
    ↓
Transformación
    ↓
Datos preparados
    ↓
Modelo de aprendizaje automático

En este capítulo construiremos nuestra primera base preparada.

2.5 Representación matemática de una base preparada

Una base de datos para aprendizaje automático puede representarse como una matriz de variables predictoras (X) y una variable respuesta (Y).

La matriz de variables predictoras es:

\[ X = \left[ \begin{array}{cccc} x_{11} & x_{12} & \cdots & x_{1p} \\ x_{21} & x_{22} & \cdots & x_{2p} \\ \vdots & \vdots & \ddots & \vdots \\ x_{n1} & x_{n2} & \cdots & x_{np} \end{array} \right] \]

y la variable respuesta es:

\[ Y = \left[ \begin{array}{c} y_1 \\ y_2 \\ \vdots \\ y_n \end{array} \right] \]

En nuestro caso, (Y) representa la variable:

accidente_con_victimas

con dos posibles valores:

Con víctimas
Solo daños

Cuando convertimos esta variable en forma binaria, podemos escribir:

[ Y_i = \[\begin{cases} 1, & \text{si el accidente tiene víctimas} \\ 0, & \text{si el accidente es de solo daños} \end{cases}\]

]

Esta codificación será útil para modelos como la regresión logística.

2.6 Algoritmo general de preparación de datos

Algoritmo: Preparación de datos para aprendizaje automático

Entrada:
  - Base de datos cruda
  - Diccionario de variables
  - Pregunta de investigación o predicción

Proceso:
  1. Leer la base de datos.
  2. Revisar dimensiones, nombres y tipos de variables.
  3. Seleccionar variables relevantes.
  4. Identificar la variable respuesta.
  5. Transformar variables cuando sea necesario.
  6. Convertir variables categóricas al tipo adecuado.
  7. Revisar datos faltantes.
  8. Eliminar o imputar registros incompletos.
  9. Guardar una base preparada.

Salida:
  - Base limpia y lista para análisis exploratorio y modelado

La preparación de datos no es un paso secundario. Es una parte central del proceso, porque la calidad del modelo depende directamente de la calidad de los datos.

2.7 Conjunto de datos de trabajo: ATUS

En este capítulo comenzaremos a trabajar con un conjunto de datos real de México: la estadística de Accidentes de Tránsito Terrestre en Zonas Urbanas y Suburbanas, conocida como ATUS, publicada por INEGI.

ATUS contiene información sobre accidentes de tránsito ocurridos en zonas urbanas y suburbanas. La información tiene cobertura nacional, estatal y municipal, y considera accidentes en caminos de jurisdicción no federal.

Este conjunto de datos es útil para construir ejemplos de aprendizaje automático porque contiene variables como:

  • entidad federativa,
  • municipio,
  • año,
  • mes,
  • día de la semana,
  • hora,
  • tipo de accidente,
  • causa presunta,
  • clase de accidente,
  • víctimas.

Con estos datos podemos formular problemas como:

¿Podemos clasificar si un accidente tendrá víctimas o será solo de daños materiales?

Ese será nuestro primer problema práctico.

2.17 Crear una variable respuesta

Primero convertimos CLASACC a texto para evitar problemas si viene como número o como etiqueta.

Code 
if (exists("atus_base") && "CLASACC" %in% names(atus_base)) {
  atus_modelo <- atus_base |>
    mutate(
      CLASACC_TXT = as.character(CLASACC),
      CLASACC_TXT = str_to_lower(CLASACC_TXT),
      accidente_con_victimas = case_when(
        str_detect(CLASACC_TXT, "sólo daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "solo daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daã") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daÃ") ~ "Solo daños",
        str_detect(CLASACC_TXT, "no fatal") ~ "Con víctimas",
        str_detect(CLASACC_TXT, "fatal") ~ "Con víctimas",
        CLASACC_TXT %in% c("1", "2") ~ "Con víctimas",
        CLASACC_TXT %in% c("3") ~ "Solo daños",
        TRUE ~ NA_character_
      )
    )
}

2.18 Nota importante sobre la codificación

Cuando se trabaja con bases reales, las categorías pueden venir codificadas de distintas formas.

Por ejemplo, una variable puede venir como:

Fatal
No fatal
Solo daños

o como códigos:

1, 2, 3

Por eso, siempre se debe consultar la documentación de la base de datos antes de modelar.

En este libro iremos documentando las transformaciones para que el análisis sea claro y reproducible.

2.20 Convertir variables categóricas

Algunas variables deben tratarse como categorías y no como números.

Por ejemplo, la entidad federativa y el municipio son códigos, pero no representan cantidades numéricas.

No tiene sentido decir que la entidad 20 es “el doble” de la entidad 10.

Por eso convertimos algunas variables a factor.

Code 
if (exists("atus_modelo")) {
  atus_modelo <- atus_modelo |>
    mutate(
      ID_ENTIDAD = as.factor(ID_ENTIDAD),
      ID_MUNICIPIO = as.factor(ID_MUNICIPIO),
      MES = as.factor(MES),
      DIASEMANA = as.factor(DIASEMANA),
      TIPACCID = as.factor(TIPACCID),
      CAUSAACCI = as.factor(CAUSAACCI),
      accidente_con_victimas = as.factor(accidente_con_victimas)
    )

  str(atus_modelo)
}

## Introducción

En los libros y cursos introductorios es común usar bases de datos pequeñas, limpias y listas para modelar. Sin embargo, en la práctica profesional los datos reales casi nunca vienen preparados.

Antes de aplicar cualquier algoritmo de aprendizaje automático, normalmente tenemos que realizar varias tareas:

- descargar o importar datos,
- revisar nombres de variables,
- corregir tipos de datos,
- detectar valores faltantes,
- seleccionar variables útiles,
- transformar variables,
- crear una variable respuesta,
- eliminar registros incompletos,
- documentar cada paso.

Esta etapa se conoce como **preparación de datos**.

En muchos proyectos reales, preparar los datos toma más tiempo que entrenar el modelo.

## ¿Por qué es tan importante preparar los datos?

Un modelo de aprendizaje automático no puede corregir por sí solo todos los problemas de una base de datos.

Si los datos están mal preparados, el modelo puede producir resultados equivocados, aunque el algoritmo sea sofisticado.

Una frase común en ciencia de datos es:

```text
Basura entra, basura sale.
```

Esto significa que si usamos datos mal organizados, incompletos o incorrectos, el resultado del modelo también será poco confiable.

## Datos crudos y datos preparados

Llamaremos **datos crudos** a la base tal como se obtiene de la fuente original.

Los datos crudos son valiosos porque contienen la información original, pero no siempre están listos para aplicar aprendizaje automático.

Llamaremos **datos preparados** a la base que ya fue revisada, transformada y organizada para un propósito específico.

La diferencia puede verse así:

```text
Datos crudos
    ↓
Revisión
    ↓
Limpieza
    ↓
Selección de variables
    ↓
Transformación
    ↓
Datos preparados
    ↓
Modelo de aprendizaje automático
```

En este capítulo construiremos nuestra primera base preparada.

## Representación matemática de una base preparada

Una base de datos para aprendizaje automático puede representarse como una matriz de variables predictoras \(X\) y una variable respuesta \(Y\).

La matriz de variables predictoras es:

$$
X =
\left[
\begin{array}{cccc}
x_{11} & x_{12} & \cdots & x_{1p} \\
x_{21} & x_{22} & \cdots & x_{2p} \\
\vdots & \vdots & \ddots & \vdots \\
x_{n1} & x_{n2} & \cdots & x_{np}
\end{array}
\right]
$$

y la variable respuesta es:

$$
Y =
\left[
\begin{array}{c}
y_1 \\
y_2 \\
\vdots \\
y_n
\end{array}
\right]
$$

En nuestro caso, \(Y\) representa la variable:

```text
accidente_con_victimas
```

con dos posibles valores:

```text
Con víctimas
Solo daños
```

Cuando convertimos esta variable en forma binaria, podemos escribir:

\[
Y_i =
\begin{cases}
1, & \text{si el accidente tiene víctimas} \\
0, & \text{si el accidente es de solo daños}
\end{cases}
\]

Esta codificación será útil para modelos como la regresión logística.

## Algoritmo general de preparación de datos

```text
Algoritmo: Preparación de datos para aprendizaje automático

Entrada:
  - Base de datos cruda
  - Diccionario de variables
  - Pregunta de investigación o predicción

Proceso:
  1. Leer la base de datos.
  2. Revisar dimensiones, nombres y tipos de variables.
  3. Seleccionar variables relevantes.
  4. Identificar la variable respuesta.
  5. Transformar variables cuando sea necesario.
  6. Convertir variables categóricas al tipo adecuado.
  7. Revisar datos faltantes.
  8. Eliminar o imputar registros incompletos.
  9. Guardar una base preparada.

Salida:
  - Base limpia y lista para análisis exploratorio y modelado
```

La preparación de datos no es un paso secundario. Es una parte central del proceso, porque la calidad del modelo depende directamente de la calidad de los datos.

## Crear una variable respuesta

Primero convertimos `CLASACC` a texto para evitar problemas si viene como número o como etiqueta.

```{r}
if (exists("atus_base") && "CLASACC" %in% names(atus_base)) {
  atus_modelo <- atus_base |>
    mutate(
      CLASACC_TXT = as.character(CLASACC),
      CLASACC_TXT = str_to_lower(CLASACC_TXT),
      accidente_con_victimas = case_when(
        str_detect(CLASACC_TXT, "sólo daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "solo daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daños") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daã") ~ "Solo daños",
        str_detect(CLASACC_TXT, "daÃ") ~ "Solo daños",
        str_detect(CLASACC_TXT, "no fatal") ~ "Con víctimas",
        str_detect(CLASACC_TXT, "fatal") ~ "Con víctimas",
        CLASACC_TXT %in% c("1", "2") ~ "Con víctimas",
        CLASACC_TXT %in% c("3") ~ "Solo daños",
        TRUE ~ NA_character_
      )
    )
}
```

Revisamos la nueva variable.

```{r}
if (exists("atus_modelo")) {
  table(atus_modelo$accidente_con_victimas, useNA = "ifany")
}
```

## Convertir variables categóricas

Algunas variables deben tratarse como categorías y no como números.

Por ejemplo, la entidad federativa y el municipio son códigos, pero no representan cantidades numéricas.

No tiene sentido decir que la entidad 20 es “el doble” de la entidad 10.

Por eso convertimos algunas variables a factor.

```{r}
if (exists("atus_modelo")) {
  atus_modelo <- atus_modelo |>
    mutate(
      ID_ENTIDAD = as.factor(ID_ENTIDAD),
      ID_MUNICIPIO = as.factor(ID_MUNICIPIO),
      MES = as.factor(MES),
      DIASEMANA = as.factor(DIASEMANA),
      TIPACCID = as.factor(TIPACCID),
      CAUSAACCI = as.factor(CAUSAACCI),
      accidente_con_victimas = as.factor(accidente_con_victimas)
    )

  str(atus_modelo)
}
```
