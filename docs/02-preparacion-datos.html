3.1 Objetivos del capítulo

Al finalizar este capítulo, el lector será capaz de:

  • Comprender por qué la preparación de datos es una etapa fundamental en aprendizaje automático.
  • Leer un conjunto de datos real en R.
  • Revisar nombres de variables, dimensiones y estructura de una base.
  • Seleccionar variables de interés.
  • Crear una variable respuesta para un problema de clasificación.
  • Preparar una primera base de trabajo lista para análisis exploratorio y modelado.

3.2 2.1 Introducción

En los libros y cursos introductorios es común usar bases de datos pequeñas, limpias y listas para modelar. Sin embargo, en la práctica profesional los datos reales casi nunca vienen preparados.

Antes de aplicar cualquier algoritmo de aprendizaje automático, normalmente tenemos que realizar varias tareas:

  • descargar o importar datos,
  • revisar nombres de variables,
  • corregir tipos de datos,
  • detectar valores faltantes,
  • seleccionar variables útiles,
  • transformar variables,
  • crear una variable respuesta,
  • eliminar registros incompletos,
  • documentar cada paso.

Esta etapa se conoce como preparación de datos.

En muchos proyectos reales, preparar los datos toma más tiempo que entrenar el modelo.

3.3 2.2 Conjunto de datos de trabajo: ATUS de INEGI

En este capítulo comenzaremos a trabajar con un conjunto de datos real de México: la estadística de Accidentes de Tránsito Terrestre en Zonas Urbanas y Suburbanas, conocida como ATUS, publicada por INEGI.

ATUS contiene información sobre accidentes de tránsito ocurridos en zonas urbanas y suburbanas. La información tiene cobertura nacional, estatal y municipal, y considera accidentes en caminos de jurisdicción no federal.

Este conjunto de datos es útil para construir ejemplos de aprendizaje automático porque contiene variables como:

  • entidad federativa,
  • municipio,
  • año,
  • mes,
  • día de la semana,
  • hora,
  • tipo de accidente,
  • causa presunta,
  • clase de accidente,
  • víctimas.

Con estos datos podemos formular problemas como:

¿Podemos clasificar si un accidente tendrá víctimas o será solo de daños materiales?

Ese será nuestro primer problema práctico.

3.13 2.12 Crear una variable respuesta

Primero convertimos CLASACC a texto para evitar problemas si viene como número o como etiqueta.

Code 
atus_modelo <- atus_base |>
  mutate(
    CLASACC_TXT = as.character(CLASACC),
    CLASACC_TXT = str_to_lower(CLASACC_TXT)
  )

Ahora creamos la variable respuesta.

Code 
atus_modelo <- atus_modelo |>
  mutate(
    accidente_con_victimas = case_when(
      str_detect(CLASACC_TXT, "fatal") ~ "Con víctimas",
      str_detect(CLASACC_TXT, "no fatal") ~ "Con víctimas",
      str_detect(CLASACC_TXT, "dañ") ~ "Solo daños",
      CLASACC_TXT %in% c("1", "2") ~ "Con víctimas",
      CLASACC_TXT %in% c("3") ~ "Solo daños",
      TRUE ~ NA_character_
    )
  )

Revisamos la nueva variable.

Code 
table(atus_modelo$accidente_con_victimas, useNA = "ifany")

Con víctimas         <NA> 
       68108       322519

3.14 2.13 Nota importante sobre la codificación

Cuando se trabaja con bases reales, las categorías pueden venir codificadas de distintas formas.

Por ejemplo, una variable puede venir como:

Fatal
No fatal
Solo daños

o como códigos:

1, 2, 3

Por eso, siempre se debe consultar la documentación de la base de datos antes de modelar.

En este libro iremos documentando las transformaciones para que el análisis sea claro y reproducible.

atus_modelo <- atus_modelo |>

3.16 2.15 Convertir variables categóricas

Algunas variables deben tratarse como categorías y no como números.

Por ejemplo, la entidad federativa y el municipio son códigos, pero no representan cantidades numéricas.

Code 
atus_modelo <- atus_modelo |>
  mutate(
    ID_ENTIDAD = as.factor(ID_ENTIDAD),
    ID_MUNICIPIO = as.factor(ID_MUNICIPIO),
    MES = as.factor(MES),
    DIASEMANA = as.factor(DIASEMANA),
    TIPACCID = as.factor(TIPACCID),
    CAUSAACCI = as.factor(CAUSAACCI),
    accidente_con_victimas = as.factor(accidente_con_victimas)
  )

## 2.11 Revisión de la variable `CLASACC`

if ("CLASACC" %in% names(atus_base)) {

```text
Con víctimas
Solo daños
```

Esta será nuestra primera variable respuesta para clasificación.

## 2.12 Crear una variable respuesta

Primero convertimos `CLASACC` a texto para evitar problemas si viene como número o como etiqueta.

```{r}
atus_modelo <- atus_base |>
  mutate(
    CLASACC_TXT = as.character(CLASACC),
    CLASACC_TXT = str_to_lower(CLASACC_TXT)
  )
```

Ahora creamos la variable respuesta.

```{r}
atus_modelo <- atus_modelo |>
  mutate(
    accidente_con_victimas = case_when(
      str_detect(CLASACC_TXT, "fatal") ~ "Con víctimas",
      str_detect(CLASACC_TXT, "no fatal") ~ "Con víctimas",
      str_detect(CLASACC_TXT, "dañ") ~ "Solo daños",
      CLASACC_TXT %in% c("1", "2") ~ "Con víctimas",
      CLASACC_TXT %in% c("3") ~ "Solo daños",
      TRUE ~ NA_character_
    )
  )
```

Revisamos la nueva variable.

```{r}
table(atus_modelo$accidente_con_victimas, useNA = "ifany")
```

## 2.13 Nota importante sobre la codificación

Cuando se trabaja con bases reales, las categorías pueden venir codificadas de distintas formas.

Por ejemplo, una variable puede venir como:

```text
Fatal
No fatal
Solo daños
```

o como códigos:

```text
1, 2, 3
```

Por eso, siempre se debe consultar la documentación de la base de datos antes de modelar.

En este libro iremos documentando las transformaciones para que el análisis sea claro y reproducible.

## 2.15 Convertir variables categóricas

Algunas variables deben tratarse como categorías y no como números.

Por ejemplo, la entidad federativa y el municipio son códigos, pero no representan cantidades numéricas.

```{r}
atus_modelo <- atus_modelo |>
  mutate(
    ID_ENTIDAD = as.factor(ID_ENTIDAD),
    ID_MUNICIPIO = as.factor(ID_MUNICIPIO),
    MES = as.factor(MES),
    DIASEMANA = as.factor(DIASEMANA),
    TIPACCID = as.factor(TIPACCID),
    CAUSAACCI = as.factor(CAUSAACCI),
    accidente_con_victimas = as.factor(accidente_con_victimas)
  )
```

Revisamos la estructura.
